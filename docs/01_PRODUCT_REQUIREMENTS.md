# 云程智绘图——产品需求说明书 V1.0

## 一、项目定位

### 1.1 产品定位

云程智绘图是一个面向求职作品展示的国内智能旅行规划系统。系统以用户提交的国内旅行需求为起点，结合用户偏好、预算约束、交通方案、酒店方案、景点与餐饮信息、天气、地图路线、本地知识库和外部检索信息，生成可比较、可编辑、可保存、可导出的结构化旅行方案。

### 1.2 项目基础

- 当前正式仓库为 `cloud-trip-agent`。
- 产品和代码基础来自现有智旅云图项目。
- 现有能力包括 FastAPI 后端、Vue 3 前端、SQLite 保存、Redis 缓存、RAG 检索、高德地图、天气查询、自然语言编辑、历史行程和 PDF 导出。
- 后续演进需要在不破坏现有可运行功能的前提下补充登录、权限、LangGraph 编排、多 Agent、版本管理、长期记忆、Tavily 检索和人工确认能力。

### 1.3 参考项目边界

`agent-ctrip-reference` 仅作为以下能力的设计参考：

- JWT 登录认证；
- LangGraph 工作流；
- 多 Agent 编排；
- 人工确认机制。

不允许直接拼接两个项目，不允许整体复制参考项目的代码、目录结构或业务实现。

## 二、第一版支持范围

V1.0 支持以下功能：

1. 用户注册和登录。
2. 国内旅行规划。
3. 用户填写出发地、目的地、日期、人数和预算。
4. 生成航班和高铁参考方案。
5. 生成酒店参考方案。
6. 生成景点和门票参考信息。
7. 生成餐饮推荐。
8. 查询和展示高德地图路线。
9. 查询和展示天气信息。
10. 通过 Tavily 检索外部公开信息。
11. 默认生成 2 个候选行程，最多生成 3 个候选行程。
12. 使用确定性代码进行预算计算。
13. 对行程进行预算、路线、时间和字段校验。
14. 支持局部重规划。
15. 支持自然语言修改行程。
16. 支持拖拽修改行程中的日程、景点、餐饮或交通顺序。
17. 支持行程保存。
18. 支持行程版本管理。
19. 支持长期记忆开关、查看和删除。
20. 支持 Agent 执行过程展示。
21. 支持 PDF 导出。

## 三、第一版明确不支持

V1.0 明确不支持以下能力：

1. Browser 自动化。
2. Playwright 操作第三方网站。
3. 携程实时网页查询。
4. 第三方网站自动登录。
5. 自动预订。
6. 自动支付。
7. 自动退票、改签和取消。
8. 自驾。
9. 12306。
10. 多平台实时比价。
11. 多人协作。
12. 管理员后台。
13. 手机 App。
14. 公网部署。

系统不得以任何文案、字段名或界面提示暗示 V1.0 已支持上述能力。

## 四、数据来源规则

### 4.1 必须标记来源的数据

航班、高铁、酒店和门票信息必须标记数据来源。允许的数据来源类型如下：

- 本地演示数据；
- 规则估算；
- 用户录入；
- Tavily 外部检索；
- 正式 API。

### 4.2 禁止性描述

不得将以下内容描述为实时价格、实时库存或可预订结果：

- 本地演示数据；
- 规则估算结果；
- 大模型生成内容；
- Tavily 搜索摘要；
- 未经正式 API 验证的数据。

### 4.3 数据入库前校验

所有进入正式行程结构的数据必须经过：

- 字段完整性校验；
- 类型校验；
- 日期和时间合法性校验；
- 金额非负校验；
- 重复项去重；
- 数据来源标记；
- 外部检索信息可信度提示。

### 4.4 确定性计算范围

以下内容必须使用确定性 Python 代码处理，不交给大模型自由计算：

- 预算；
- 日期；
- 权限；
- 版本号；
- 行程状态机；
- 最大候选方案数量；
- 最大重规划次数；
- 最大重试次数。

## 五、Tavily 使用范围

### 5.1 允许用途

Tavily 主要用于检索公开、辅助性的旅行信息：

- 景点公开信息；
- 开放时间；
- 餐饮信息；
- 旅游提示；
- 公开攻略；
- 临时政策或注意事项。

### 5.2 禁止用途

Tavily 不用于：

- 获取或声称获取携程实时价格；
- 获取或声称获取实时库存；
- 生成可预订结果；
- 自动访问订单页面；
- 登录第三方网站；
- 提交订单。

### 5.3 调用边界

- Tavily 只能通过后端封装的 Tavily 工具调用。
- 不允许 Agent 自由访问任意网址。
- Tavily API Key 只能保存在后端 `.env` 中。
- Tavily API Key 不得输出到日志、前端或模型上下文。
- Tavily 调用失败时，应回退到高德 POI、本地知识库或演示数据。

### 5.4 结果保存字段

每条 Tavily 搜索结果必须保存：

- 标题；
- 来源网址；
- 摘要；
- 查询时间；
- 数据来源类型。

Tavily 结果属于外部检索信息，不能直接视为确定事实。写入行程前必须经过字段校验、去重和来源标记。

## 六、主要用户流程

```text
注册登录
→ 填写旅行表单
→ 读取用户偏好
→ 生成任务计划
→ 多 Agent 执行
→ 高德、天气、RAG、Tavily和本地数据查询
→ 生成2至3个候选行程
→ 校验预算、路线和时间
→ 局部重规划
→ 用户修改
→ 保存版本
→ 导出 PDF
```

### 6.1 流程说明

1. 用户注册或登录后进入工作台。
2. 用户在新建行程页填写出发地、目的地、日期、人数、预算和偏好。
3. 系统读取用户长期偏好；若用户关闭长期记忆，则本次规划不读取历史偏好。
4. Requirement Agent 将用户输入转换为结构化需求。
5. Supervisor Agent 生成任务计划并分发给各专业 Agent。
6. 各 Agent 通过白名单后端工具查询高德、天气、RAG、Tavily 和本地数据。
7. Planner Agent 与 Itinerary Builder 汇总生成默认 2 个、最多 3 个候选行程。
8. Verification Agent 校验预算、时间、路线、字段完整性和来源标记。
9. 校验失败时，Replanner Agent 只对失败部分进行局部重规划。
10. 用户可通过自然语言或拖拽修改行程。
11. 每次保存生成一个新版本。
12. 用户可导出当前版本为 PDF。

## 七、页面需求

### 7.1 登录注册页

- 支持用户注册。
- 支持用户登录。
- 登录成功后进入工作台。
- 注册和登录失败时展示明确错误信息。

### 7.2 工作台

- 展示最近行程。
- 提供新建行程入口。
- 提供历史行程入口。
- 提供长期记忆管理入口。

### 7.3 新建行程页

- 支持填写出发地、目的地、开始日期、结束日期、人数和预算。
- 支持填写旅行偏好、饮食偏好、住宿偏好和特殊要求。
- 提交前必须做前端基础校验。

### 7.4 Agent 任务执行页

- 展示任务计划。
- 展示各 Agent 的执行状态。
- 展示工具调用名称、状态、耗时和失败原因。
- 不展示 API Key、Token 或敏感环境变量。

### 7.5 候选方案页

- 默认展示 2 个候选方案。
- 最多展示 3 个候选方案。
- 每个方案展示预算、天数、交通摘要、酒店摘要、核心景点和数据来源提示。
- 不得将候选方案描述为可预订订单。

### 7.6 行程编辑工作区

- 支持自然语言修改。
- 支持拖拽调整日程、景点、餐饮或交通顺序。
- 修改后必须触发预算、时间和路线校验。
- 保存后生成新版本。

### 7.7 历史行程页

- 展示已保存行程列表。
- 支持按创建时间或更新时间查看。
- 支持打开行程详情。
- 支持删除用户自己的行程。

### 7.8 版本管理页

- 展示同一行程的版本列表。
- 展示版本号、创建时间、修改来源和摘要。
- 支持查看任意历史版本。
- 支持从历史版本恢复为新版本。

### 7.9 长期记忆管理页

- 支持开启或关闭长期记忆。
- 支持查看已保存偏好。
- 支持删除单条偏好。
- 支持清空当前用户的长期记忆。

## 八、Agent 需求

### 8.1 Agent 与工具清单

V1.0 至少包含：

- Supervisor Agent；
- Requirement Agent；
- Planner Agent；
- Transport Agent；
- Hotel Agent；
- POI Agent；
- Ticket Agent；
- Food Agent；
- Weather Agent；
- Tavily Search Tool；
- Budget Engine；
- Itinerary Builder；
- Verification Agent；
- Replanner Agent。

### 8.2 使用大模型的任务

以下任务可以使用大模型：

- 将用户自然语言需求转换为结构化需求草稿；
- 识别用户偏好和约束；
- 生成候选行程叙述和方案差异说明；
- 根据已校验数据生成自然语言摘要；
- 根据用户自然语言修改要求生成局部修改建议；
- 对外部检索摘要进行旅行语境下的归纳。

大模型输出不得直接写入最终行程，必须经过 Pydantic 或等价结构校验、来源标记和确定性规则校验。

### 8.3 必须使用确定性 Python 代码的任务

以下任务必须使用确定性 Python 代码：

- JWT 鉴权和用户权限判断；
- 用户数据隔离；
- 日期范围计算；
- 天数计算；
- 预算拆分和合计；
- 版本号生成；
- 行程状态机流转；
- 候选方案数量限制；
- 最大重试次数控制；
- 最大重规划次数控制；
- 字段校验；
- 去重；
- 数据来源标记；
- API Key 读取和保护；
- 工具白名单校验；
- PDF 导出。

### 8.4 工具白名单

Agent 访问外部信息时，只能调用经过白名单限制的后端工具，例如：

- TavilySearchTool；
- AmapPOITool；
- WeatherTool；
- LocalRAGTool；
- RouteTool；
- BudgetTool。

不允许给 Agent 提供以下能力：

- Shell；
- PowerShell；
- subprocess；
- eval；
- exec；
- 任意 Python 执行；
- 任意文件读写；
- 任意网址直接访问；
- Docker API。

## 九、非功能需求

### 9.1 用户数据隔离

- 每个用户只能访问自己的行程、版本和长期记忆。
- 后端接口必须从认证上下文识别用户身份。
- 不允许前端传入 `user_id` 后直接作为可信身份。

### 9.2 密钥安全

- API Key、密码、Token 只能保存在后端安全配置中。
- `.env` 不允许提交。
- 日志、前端响应和模型上下文不得包含真实密钥。

### 9.3 工具白名单

- Agent 只能调用后端注册的白名单工具。
- 工具必须限制输入字段和调用目标。
- Tavily 工具不得开放任意网址抓取能力。

### 9.4 Agent 失败降级

- Tavily 失败时回退到高德 POI、本地知识库或演示数据。
- 高德 POI 失败时回退到本地知识库或演示数据。
- 天气失败时保留行程生成，但标记天气信息不可用。
- LLM 调用失败时应返回可解释错误或规则生成结果。

### 9.5 重试和重规划限制

- 单个外部工具默认最多重试 1 次。
- 单次行程生成最多触发 2 次局部重规划。
- 超过限制后必须返回失败原因和可恢复建议。

### 9.6 日志与可观测性

- 记录请求 ID、用户 ID、行程 ID、版本号、Agent 名称、工具名称、执行状态、耗时和失败原因。
- 不记录 API Key、密码、Token 或完整敏感输入。
- Agent 执行过程应可在前端展示摘要。

### 9.7 数据来源可追溯

- 航班、高铁、酒店、门票、景点、餐饮和外部检索信息必须带来源。
- 前端必须展示关键数据来源提示。
- PDF 导出必须保留数据来源说明。

### 9.8 Docker 本地运行

- V1.0 必须支持 Docker 本地运行。
- 至少包含前端、后端和 Redis 服务。
- 本地运行不等同于公网部署。

### 9.9 Agent 安全边界

- 不允许 Agent 执行系统命令。
- 不允许 Agent 进行任意文件操作。
- 不允许 Agent 直接访问 Docker API。
- 不允许 Agent 访问未经过后端白名单封装的外部网址。

## 十、验收标准

### 10.1 注册登录

- 使用未注册用户名注册时，系统创建用户记录并返回注册成功。
- 使用已存在用户名再次注册时，系统返回明确错误。
- 使用正确用户名和密码登录时，系统返回有效认证凭证。
- 使用错误密码登录时，系统返回登录失败，且不返回内部异常堆栈。
- 未登录用户访问受保护的行程接口时，系统返回未认证错误。
- 密码必须经过安全哈希后存储，不允许明文保存。

### 10.2 行程生成

- 用户填写出发地、目的地、日期、人数和预算后，可以生成候选行程。
- 默认返回 2 个候选行程。
- 系统不得返回超过 3 个候选行程。
- 每个候选行程必须包含每日安排、交通摘要、酒店摘要、餐饮建议、预算明细和数据来源字段。
- 预算总额必须由后端确定性代码计算。

### 10.3 候选方案差异

- 两个默认候选方案在至少 2 个维度上存在差异，维度可包括交通方式、酒店区域、景点顺序、预算分配或行程节奏。
- 候选方案页必须展示差异摘要。
- 差异摘要不得声称实时价格或实时库存。

### 10.4 Tavily 检索

- 触发 Tavily 检索后，每条结果保存标题、来源网址、摘要、查询时间和数据来源类型。
- Tavily 结果写入行程前必须带 `Tavily 外部检索` 来源标记。
- Tavily API Key 不出现在后端日志、前端响应和模型上下文中。
- Tavily 调用失败时，系统仍可回退到高德 POI、本地知识库或演示数据。
- Tavily 搜索结果不得显示为携程实时价格、实时库存或可预订结果。

### 10.5 地图和天气

- 高德 POI 成功时，景点至少包含名称、地址、经纬度和来源标记。
- 路线查询成功时，交通段至少包含起点、终点、距离或预计耗时。
- 天气查询成功时，行程页展示目的地天气摘要。
- 天气查询失败时，行程页展示天气不可用提示，且行程主体仍可展示。

### 10.6 修改和版本

- 用户通过自然语言修改行程后，系统生成一个新版本。
- 用户拖拽修改行程后，系统生成一个新版本。
- 每个版本必须包含版本号、创建时间、修改来源和行程快照。
- 用户可以查看历史版本。
- 用户可以从历史版本恢复，并生成新的当前版本。

### 10.7 长期记忆

- 用户可以开启长期记忆。
- 用户可以关闭长期记忆。
- 关闭长期记忆后，新建行程不得读取历史偏好。
- 用户可以查看当前保存的偏好。
- 用户可以删除单条偏好。
- 用户可以清空自己的长期记忆。

### 10.8 Agent 编排

- 每次生成行程时，前端可以展示 Supervisor Agent 生成的任务计划。
- 前端可以展示至少 5 类 Agent 或工具执行状态。
- 每个 Agent 执行记录至少包含名称、状态、开始时间、结束时间或耗时。
- Agent 调用失败时，前端展示失败节点和降级结果。

### 10.9 错误降级

- Tavily 失败时，系统回退到高德 POI、本地知识库或演示数据。
- 高德失败时，系统回退到本地知识库或演示数据。
- LLM 失败时，系统返回规则生成结果或明确失败原因。
- 超过最大重规划次数后，系统停止重规划并返回失败原因。
- 所有降级结果必须保留数据来源标记。

### 10.10 PDF 导出

- 用户可以导出当前行程版本为 PDF。
- PDF 至少包含目的地、日期、每日行程、预算明细和数据来源说明。
- PDF 导出不得包含 API Key、Token 或用户密码。
- PDF 中不得将演示数据、模型生成内容或 Tavily 摘要称为实时价格。
